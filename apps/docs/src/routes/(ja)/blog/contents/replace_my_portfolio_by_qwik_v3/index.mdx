こんにちは。[@silverbirder](https://x.com/silverbirder)です。
最近、私のポートフォリオページとブログページを Qwik フレームワークでリニューアルしました。これで、全体のリニューアルは 4 回目になります。この記事では、その過程で得た学びや経験を共有します。

ちなみに、これまでのリニューアルに関する記事は、`Portfolio`タグで検索してみてください。また、GitHub のリリースタグも利用していますので、以下のリンクから過去のソースコードを参照できます。

[https://github.com/silverbirder/silverbirder.github.io/tags](https://github.com/silverbirder/silverbirder.github.io/tags)

## なぜ刷新したのか

刷新の動機は主に好奇心です。それに続いて、自分の技術的知見を広げるためでもあります。過去には静的サイトジェネレータの Hugo や Google の AMP、Rocket などを試してきました。それぞれの技術には当時の流行や興味を反映していました。今回も、現在のトレンドや興味を持っている技術を中心に選択しました。

## Qwik フレームワークの選択

Qwik はパフォーマンスを重視したフレームワークですが、私がそれを選んだ主な理由は Markdown のサポートが充実しているからです。さらに、`State of JS`での高い評価や、React に近い構文も魅力的でした。ブログの Markdown にはタグ情報をメタデータとして持っていたので、それを活用したいと考えました。そんな中、[https://github.com/doganozturk/blog-qwik](https://github.com/doganozturk/blog-qwik) というリポジトリを発見しました。これを参考にブログページの構築を開始しました。

## 学んだこと

Qwik を使用することで、フレームワーク自体だけでなく、関連する技術やツールにも触れることができました。

### PandaCSS

例えば、PandaCSS というものがありました。私は、これまで Bootstrap や materialUI のような CSS フレームワークを
使うことが多かったのです。
けど、最近業務でランディングページの開発をする経験から、CSS を書いてみたくなりました。
そこで、PandaCSS を使って、自分で CSS を書いていきました。

もっと、こうすべきだという手段もあるのですが、まずは使ってみることです。
次に、テキストやスペーシング、カラーなどを定義していき、それらを使い回すようにやろうと思いました。
最初から、primary や secondary、テキストのよく使うもの、スペーシング、レイアウト、などもそうです。
これらを PandaCSS で定義し、使っていきたいなと思ってます。

### 多言語対応

多言語対応も、初めてチャレンジしました。といってもなんちゃってです。
多言語対応する場合、Markdown の日本語記事とトップページなどにある日本語の文言をどうしたら良いかとなりました。

本来の運用としては、日本語の記事を Push した後、英語に詳しい人が翻訳するのが大筋かなと思います。
Localization の SaaS を調べて、それを使うのもありかなと思いました。
が、個人レベルなら、そこまでやる必要はないかなと思いました。
そこで、qwik-speack、chatgpt-md-translator の 2 つを使って翻訳しました。
内部では、OpenAI の使っているようです。

多言語対応をすると、Push 通知するときやリンクするコンテンツにも気を使うようになりました。
実際に業務として使うならば、最初の段階からあると良いかもなぁと思います。
もちろん、後から埋めることも可能だと思います。

### SaaS の利用

今回は、SaaS を色々使ってみました。

#### OpenReplay

例えば、OpenReplay というものを導入しました。
これは、ユーザーの行動を監視するものです。
実際の運用としては、トラブルシュートや UX 改善などに使えそうなものに思いました。

これをブログページに入れたので、どのページのどのあたりにマウスがあるのかをみれて、リンク配置や動線を考えさせられました。

#### OneSignal

次に、Push 通知もやってみたいと思いました。
しかし、Push 通知するサーバー側の用意が必要と思うのですが、それを掛けるコストはなくしたいと思い、
よく知られている OneSignal というものを入れてみました。

通知をするときに、言語の選択や、通知を受け取るためのベストプラクティスが OneSignal に書いてあり、勉強になりました。

https://documentation.onesignal.com/docs/permission-requests#what-are-some-best-practices-around-web-push-prompting

#### Giscus

Qwik を選ぶ前、何を使うか調べていたときに `Material for MkDocs` というのを発見し、
その中で、giscus というを見つけました。

ブログへのコメントをもらう機能も、楽に入れられたので良いなぁと思いました。

#### その他

その他には、Buy me a coffee なんてものを入れてみたり、
lottie という軽量アニメーションを入れてみたり、知らないことを多く知る機会となりました。

## 途中で諦めたもの

### Changesets

`changesets`を利用してバージョン管理を試みましたが、途中で挫折しました。

### Turborepo

`turborepo`での monorepo 構成も夢見ていましたが、挫折しました。考えていた構成は以下の通りです：

- apps/docs
- packages/ui
- packages/blog-contents
- packages/translate

しかし、PandaCSS や Qwik の UI ライブラリをうまく組み合わせるのが難しく、実現できませんでした。

### Fuse.js

ブログの検索機能の実装も考えましたが、自分で実装するのが面倒になり、断念しました。

## 今後の展望

例えば、dark モード対応、もしくはテーマカラーの変更できる機能を作ってみたいなと思いました。
他には、Playwright を使って、cucumber を試してみたいなと思っています。
デザインは、初めて CSS でフルスクラッチでやったので、デザインにもっと強くなりたいと思いました。
さらに、vercel が v0 というのを発表しているので、これを試すのも良さそう。
また、turbo を monorepo として導入したものの、全然活かしきれていないので、キャッシュして CI を高速化してみたいですね。
bun を使って、速度が速くなるかとかも試してみたいですね。
