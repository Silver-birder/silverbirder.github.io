「このスーパーの白菜はちょっと高いかな。他に安いところはないのかな？」そんな日常の小さな疑問から、スーパーごとの食材価格を手軽に比較できるWebアプリ「ぼちぼち」を開発しました。

https://bochi-bochi.vercel.app

## アプリの使い方

1. 近くのスーパーを選択します。
2. 欲しい食材を選ぶと、どのスーパーで安く買えるかが一目瞭然です。

私の妻もこのアプリを利用してから、「白菜はここが一番安いね」「ひき肉はあそこのスーパーだね」と言って、満足しながら買い物を楽しんでいます。買い物後のレシートを使って、食材の価格情報を簡単にアプリに登録する機能も備えています。また、AI技術を活用してスーパーの食材データを自動で収集・登録する仕組みを構築しています。

スーパーの検索・表示にはGoogle Mapを活用しており、無料枠をうまく使うことで、コストを抑えつつ豊富な地図情報を提供しています。

https://mapsplatform.google.com/intl/ja/pricing/

## 全てが安いスーパーが誕生

しかし、このアプリを利用しているうちに、最終的には近所に新しくできた「全てが安いスーパー」でほとんどの買い物をするようになりました。そのため、アプリ自体の利用頻度は少なくなりましたが、笑い話として楽しんでいます。

## 開発備忘録

OpenAIをシステムに組み込む際には、全てを一度に任せるよりも、プロセスを段階的に分けると精度が向上しました。
例えば、レシート画像から構造化データを作成する際に、OCRでテキスト化した後、そのテキストをOpenAIで分析する方法が最も効果的でした。
また、『"商品名"が"商品カテゴリ"に属さない場合、属することが可能な商品カテゴリを考えて』 のような指示する内容が不明瞭なものよりも、
『"商品名"が"商品カテゴリ"に属するか否か』のような答えの選択肢を制限する、指示する内容をシンプルに保つ とすると、回答の精度もよくなりました。

また、Google Map APIを活用して地図やスーパーの情報を取得し、テキスト検索や近くの場所検索を実現しました。

技術スタックは、

- Next.js
- tRPC
- planetscale
- prisma
- tremor
- jotai
- @vis.gl/react-google-maps
- crawlee
- cloud run job

開発中は妻から直接フィードバックをもらいながら、方向性を確認して進めることができ、非常に楽しい経験でした。ただ、UI/UXデザインにはまだ改善の余地があり、今後はデジタル庁のデザインシステムを参考に、さらに洗練されたデザインを目指していきます。
